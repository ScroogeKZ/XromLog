Название проекта: Система управления заявками на отгрузку "Хром логистика"
Общее описание:
Необходимо разработать полнофункциональное веб-приложение для внутреннего использования в компании. Цель — автоматизировать и систематизировать процесс приема, обработки и отслеживания заявок на отгрузку товаров. Система должна быть интуитивно понятной для рядовых сотрудников, иметь современный, чистый дизайн и предоставлять мощные инструменты для анализа и отчетности для руководства.

Ключевые требования и функционал:
1. Технологический стек:

Фронтенд: React.js с использованием библиотеки компонентов MUI (Material-UI) для создания профессионального и адаптивного пользовательского интерфейса. Использовать React Router для навигации.

Бэкенд: Node.js с фреймворком Express.js для создания REST API.

База данных: PostgreSQL для надежного хранения данных.

Аутентификация: Простая система на основе ролей (Сотрудник, Менеджер/Логист) с использованием JWT (JSON Web Tokens).

2. Модель данных (Схема базы данных):

Создать таблицу ShipmentRequests со следующими полями:

id: SERIAL PRIMARY KEY (уникальный идентификатор, автоинкремент)

request_number: VARCHAR(255) UNIQUE NOT NULL (номер заявки, генерируется автоматически, например, AST-2025-001 или INT-2025-002)

category: VARCHAR(50) NOT NULL ('Астана' или 'Междугородний')

status: VARCHAR(50) NOT NULL DEFAULT 'Новая' (Статусы: 'Новая', 'В обработке', 'Назначен транспорт', 'В пути', 'Доставлен', 'Отменен')

created_at: TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP

updated_at: TIMESTAMP WITH TIME ZONE

created_by_user_id: INTEGER (связь с таблицей пользователей)

cargo_name: VARCHAR(255) NOT NULL (наименование груза)

cargo_weight_kg: DECIMAL(10, 2) (вес)

cargo_volume_m3: DECIMAL(10, 2) (объем)

cargo_dimensions: VARCHAR(255) (габариты ДхШхВ)

package_count: INTEGER (количество мест)

special_requirements: TEXT (особые требования)

loading_address: TEXT NOT NULL (адрес погрузки)

loading_contact_person: VARCHAR(255) (контакт на погрузке)

loading_contact_phone: VARCHAR(20) (телефон)

unloading_address: TEXT NOT NULL (адрес выгрузки)

unloading_contact_person: VARCHAR(255) (контакт на выгрузке)

unloading_contact_phone: VARCHAR(20) (телефон)

desired_shipment_datetime: TIMESTAMP WITH TIME ZONE (желаемая дата и время)

notes: TEXT (примечания)

transport_info: JSONB (для данных о машине и водителе: { "driver_name": "...", "driver_phone": "...", "vehicle_model": "...", "vehicle_plate": "..." })

3. Фронтенд (Пользовательский интерфейс):

Дизайн:

Чистый и минималистичный: Использовать светлую цветовую схему с акцентным синим или зеленым цветом.

Адаптивность: Интерфейс должен корректно отображаться на всех устройствах, от широкоформатных мониторов до смартфонов.

Навигация: Боковая панель навигации с иконками и названиями разделов: "Дашборд", "Создать заявку", "Отчеты".

Страницы:

Страница входа: Форма для ввода логина и пароля.

Главный дашборд (Таблица заявок):

Отображение всех заявок в виде таблицы с колонками: № Заявки, Статус, Категория, Адрес погрузки, Адрес выгрузки, Дата создания.

Цветовая индикация статусов: Использовать цветные "чипы" (компонент MUI Chip) для статусов для быстрой визуальной оценки.

Интерактивность: При клике на строку таблицы открывается страница с детальной информацией о заявке.

Фильтрация: Выпадающие списки для фильтрации по Статусу и Категории.

Поиск: Поле для динамического поиска по номеру заявки, адресу или наименованию груза.

Пагинация: Разделение длинного списка заявок на страницы.

Страница создания/редактирования заявки:

Интуитивно понятная форма со всеми полями из модели данных.

Разделение на логические блоки: "Информация о грузе", "Адрес погрузки", "Адрес выгрузки".

Выбор категории: Радио-кнопки для выбора "Отгрузки по Астане" или "Междугородние отгрузки".

Валидация полей: Проверка на обязательное заполнение ключевых полей (адреса, контакты, наименование груза) и корректность формата данных.

Страница просмотра заявки (Карточка заявки):

Отображение всей информации о заявке в удобном для чтения формате.

Лента статусов (трекинг): Визуальная временная шкала, показывающая историю изменения статусов с датами (например, "Новая" - 01.08.2025 14:30, "В обработке" - 01.08.2025 16:00).

Блок управления (для Менеджера/Логиста): Возможность изменить статус заявки, а также добавить/изменить информацию о назначенном транспорте.

4. Функционал для отчетности (для руководства):

Отдельная страница "Отчеты":

Дашборд руководителя: Набор виджетов с ключевыми показателями.

Виджеты:

"Статистика за период": Фильтр по дате (неделя, месяц, квартал, произвольный период). Показывает:

Общее количество созданных заявок.

Количество выполненных заявок (статус "Доставлен").

Количество отмененных заявок.

"Заявки по категориям": Круговая диаграмма, показывающая соотношение заявок "по Астане" и "Междугородних".

"Заявки по статусам": Столбчатая диаграмма, показывающая, сколько заявок находится в каждом статусе в реальном времени.

Таблица для экспорта:

Таблица со всеми данными по заявкам за выбранный период.

Кнопка "Экспорт в Excel (.xlsx)", которая генерирует и скачивает файл с данными из таблицы для дальнейшего анализа.
